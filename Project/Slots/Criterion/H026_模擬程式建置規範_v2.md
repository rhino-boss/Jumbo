## 5. 遊戲規則實作（請照順序）
1. Base spin 產生盤面（見 §11 抽樣流程）。
2. 20 paylines 連線判定（由左到右,且每線僅最高獎）。
3. Wild 替代（不可替代 Scatter）。
4. Scatter 計數與 FG 觸發。
5. 得分符號消除。
6. 金框命中處理：命中後原位轉 Wild, 並收集 multiplier（加總累積）。
7. 掉落補牌（cascade）, 重複判定直到無新連線。
8. 單次 spin 結束時套用累積倍數:
   - **BG**: spin 結束套用後清零, 下一個 BG spin 從 0 重新累積。
   - **FG**: spin 結束套用, 但整個 FG 期間不清零, 持續累積到 FG 結束（含 retrigger 延伸轉數）。
9. FG 觸發轉數: `free_spins = 15 + (num_SC - 3) * 2`, 即 3/4/5 SC → 15/17/19 場。
10. Retrigger: 同公式加場; 在「當前 FG spin 結算結束後」才把延伸轉數加入剩餘計數。
11. **Max Win 規則已取消**: 不實作 5000x 截斷; BG、FG 皆跑完整流程,不在任何層級截斷。

## 6. 邊界條件（已定案）
1. **全 Wild 連線的主 symbol 認定**: 視為「最高分符號」中獎（Wild 自身在 paytable 內為 0,不獨立賠付）。
2. **Wild 獨立賠付**: 依 xlsx paytable; 目前 xlsx 內 WW 三欄皆為 0 → 無獨立賠付。
3. **Scatter 參與連線賠分**: ❌ 不參與。
4. **Scatter 消除與掉落**:
   - Scatter 不被消除（即便與中獎連線同位置也不移除）。
   - Scatter 不進入掉落補位; 所有 `Drop_Weight_*` 中 Scatter 對應列的值視為 0。
5. **FG retrigger 加轉時機**: spin 結束後才加入剩餘轉數（不在 spin 內即時加）。
6. **R3 特殊機制**: 不寫特例邏輯; 由輪帶（strip）直接在對應位置寫入金 symbol id, 抽樣出來自然呈現「整排金框」的效果。BG 機率性 / FG 保證性差異, 由 `BG_strip*` 與 `FG_strip*` 的 Symbol 分布自行承擔。
7. **金框 → 一般 symbol 的 id 對應**: 依 xlsx paytable 實際 id 分佈計算偏移, 不硬編 `-8` 或 `-9`。當前 xlsx: M1=2/G1=11, M2=3/G2=12, ... 差值為 9, 但實作上以查表或動態推導為準。
8. **Wild 不自然掉落**: Wild 不會在 cascade 補位中被抽到; `Drop_Weight_*` 中 WW 對應列的值視為 0。Wild 只能透過金框消除轉換產生。

## 11. 倍數抽樣流程（Game Logic Spec 落地版）

### 11.1 盤面生成
1. **Table Selection**: 依 `weight` 表的 `Table_BG / Table_FG / Table_BF` 權重, 決定本次 spin 使用哪張 strip（3 張 BG / 3 張 FG / 1 張 BF）。
2. **Symbol Distribution**: 由選到的 strip 中, 按 `Strip_Weight_R1..R5` 權重滾停, 產出 3×5 盤面。

### 11.2 初始倍數判定（Initial Spin, 盤面出現金框時）
1. **Special Pool 啟動判定**:
   - 依 `Used_Special_Pool_Weight_{strip}` 該欄 + 當前盤面金框數量（0–9 對應 10 行索引）, 決定是否啟動 Special Pool。
   - **Special Pool 僅於 BG 有效**（F1–F3、BF 該欄全 0 → 永不啟動）。
2. **若 Special Pool 啟動**:
   - 隨機選 1 個金框, 倍數依 `Multiple_Selection_Weight_Special_{strip}` 抽。
   - 其餘金框, 倍數依 `Multiple_Selection_Weight_Before_{strip}` 個別抽。
3. **若 Special Pool 未啟動 / FG / BF**:
   - 所有金框倍數皆依 `Multiple_Selection_Weight_Before_{strip}` 個別抽。

### 11.3 消除掉落階段（Cascade）
1. 消除中獎符號, 金框位置原地轉 Wild, 同時將該金框倍數加總到盤面累積倍率。
2. 新掉落產生的金框, 倍數統一依 `Multiple_Selection_Weight_After_{strip}` 個別抽。
3. 重複 §11.3 直到無新連線。

### 11.4 結算
1. 當 spin 總得分 = Σ(每次 cascade 命中連線賠分) × 累積倍率值。
2. BG: 套用後清零; FG: 套用後保留累積值到下一個 FG spin。

## 12. Weight / Strip 欄位對應（資料載入契約）

### 12.1 `weight` sheet
- `Table_BG / Table_FG / Table_BF`: 各自對應 3/3/1 張 strip 的選取權重。
- `Multiple_Selection_Weight_Special_{B1,B2,B3,F1,F2,F3,BF}`: Special Pool 抽到時使用的倍數分布, 13 行對應 Multiplier_Range 的 13 檔。
- `Multiple_Selection_Weight_Before_{...}`: 初始盤面金框（Special Pool 外）的倍數分布。
- `Multiple_Selection_Weight_After_{...}`: Cascade 後新金框的倍數分布。
- `Used_Special_Pool_Weight_{B1,B2,B3,F1,F2,F3,BF}`: 10 行對應盤面金框數 0–9, 決定是否啟動 Special Pool; F1–F3 與 BF 該欄全 0。

### 12.2 `*_strip` sheets (每張 strip 共 20 欄)
- col 0–4: `Symbol_ID_R1..R5`
- col 5–9: `Strip_Weight_R1..R5`（滾停權重）
- col 10–14: `Drop_Weight_A_R1..R5`（cascade 掉落權重表 A）
- col 15–19: `Drop_Weight_B_R1..R5`（cascade 掉落權重表 B）
- 每 spin 以 50/50 隨機選 A 或 B 當作該 spin 的 cascade 掉落表。

### 12.3 Symbol id 對應（依 xlsx `pay_table`）
| id | Symbol | 類別 |
|----|--------|------|
| 0  | WW | Wild |
| 1  | C1 | Scatter |
| 2–6 | M1, M2, M3, M4, M5 | 高分 |
| 7–10 | A, K, Q, J | 低分 |
| 11–15 | G1, G2, G3, G4, G5 | 高分金框 |
| 16–19 | GA, GK, GQ, GJ | 低分金框 |

### 12.4 Paylines
- 20 條, xlsx 存為 5 位字串（如 `"01210"`）。
- 載入時轉成 `np.int64` shape (20, 5)。

### 12.5 Multiplier Range
- `value` sheet 欄 `Multiplier_Range` = `[0, 2, 3, 5, 8, 10, 15, 20, 25, 35, 50, 80, 100]` 13 檔。
- index 0 (`0`) 代表「當前未累積任何金框倍數」; 整 spin 未命中金框時, 贏分乘以 1（不做乘法）。
- index 1–12 為實際可得的倍率值。

## 13. 已確認假設清單（成品需保留此節並標示出處）

1. **Max Win 截斷**: 已取消; 不實作 5000x 強制結束（§5.11）。

6. **R3 整排金框**: 不寫特例; 由 strip 資料直接承擔（§6.6）。
7. **Drop A/B**: 每 spin 50/50 隨機（§12.2）。
8. **Special Pool 範疇**: 僅 BG, FG 與 BF 永不啟動 Special Pool（§11.2）。
9. **Used_Special_Pool_Weight 索引**: 10 行對應金框數 0–9; 單欄 scalar weight, 抽到非 0 視為啟動（§11.2、§12.1）。

11. **Feature Buy**: 僅 `mode_featurebuy`（75x）, 不實作 super_feature_buy。
12. **BF 進 FG 的轉數**: 若 BF 盤面 SC ≥ 3 依公式; SC < 3 時視為 3 SC → 15 場（保底進 FG）。
13. **金框→一般符號 id 對應**: 以 xlsx paytable 實際分佈推導（當前差值為 9）, 不硬編。
14. **Record 維度**: 沿用 H015 的「連線長度 × symbol × BG/FG」; 連線長度 = {3, 4, 5}（§8.5）。
