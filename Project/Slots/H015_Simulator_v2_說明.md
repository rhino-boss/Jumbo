# 參數使用與遊戲邏輯說明

## 文件目的

這份文件只整理兩件事：

- 參數怎麼使用
- 遊戲邏輯怎麼運作

不談檔案結構、函式名稱或程式實作位置。

## 一、主要參數

### 1. 下注倍率

`bet_multi`

用途：

- 放大所有賠付
- 同時放大單次下注成本

如果 `bet_multi = 1`，代表使用基準下注。

### 2. 下注模式

`bet_mode`

目前可用概念上有兩種：

- 一般下注
- 買免費遊戲

差異：

- 一般下注會先跑 base game，再看是否觸發 free game
- 買免費遊戲會直接進入 free game 對應的盤面邏輯

### 3. 單次下注成本

`coin_in`

用途：

- 作為 RTP、倍數、平均獎金等統計的分母

計算概念：

- 一般下注: `基礎下注 * 下注倍率`
- 買免費遊戲: `基礎下注 * 下注倍率 * 買免價格倍數`

目前規則下：

- 基礎下注 = `100`
- 買免費遊戲價格倍數 = `75`

所以買免費遊戲模式下，統計倍數會明顯受到較高成本影響。

### 4. 總模擬圈數

`total_round`

用途：

- 控制總共模擬幾個外層 round

每個 round 代表一次完整入口，可能只包含 base game，也可能包含 base game 加整場 free game。

### 5. 免費遊戲上限

`max_spin_free_game`

用途：

- 限制單次免費遊戲最多能打幾把

目前上限是：

- `50`

即使中途一直 re-trigger，也不會超過這個上限。

## 二、盤面與基本規則

### 1. 盤面大小

盤面是：

- 5 列
- 6 輪

### 2. 可計分區域

可計分區域不是完整矩形。

特性：

- 前 3 列全部有效
- 第 4 列左右角有無效格
- 第 5 列有更多無效格

因此盤面雖然看起來是 5x6，但真正參與 ways 計算的位置比較少。

### 3. 符號類型

可概括分成：

- Wild
- 觸發免費遊戲的特殊符號
- 一般可算分符號
- 金框版本的可算分符號

其中最重要的是：

- Wild 可替代一般符號
- 金框符號在命中後不會像普通符號一樣直接補新牌，而是會轉成 Wild

## 三、得分邏輯

### 1. Ways 計分

遊戲採用 ways 概念。

核心規則：

- 從最左輪開始判斷
- 只要某個符號連續出現在前幾輪，就形成 line
- 每輪命中數量相乘得到該符號的 ways 數
- 符號本身的賠付再乘上 ways 數，得到該符號本段得分

### 2. 最低成獎條件

一般可算分符號需要：

- 至少連到 3 輪

才會產生賠付。

### 3. Wild 替代

Wild 可以替代一般算分符號參與 ways 計算，因此會提高：

- 命中輪數
- ways 數量
- 總賠付

## 四、特殊玩法核心

### 1. 金框符號

金框符號在計分時，會先視為一般符號參與計算。

但如果它被命中，後續處理不是直接消失補牌，而是：

- 該位置保留特殊價值
- 下一步轉成 Wild

這會讓後續 cascade 更容易延續。

### 2. 閃電與乘數

每次 cascade 都會抽這一段要出現幾個閃電。

閃電本身的作用不是直接給分，而是：

- 推進乘數等級

乘數等級推進後，會對應到一個實際乘數值，套用在當前這段 cascade 的得分上。

### 3. 乘數等級上限

乘數等級有封頂。

目前上限是：

- `10`

## 五、Base Game 邏輯

### 1. 進場流程

每次 base game 開始時，會先決定：

- 使用哪組 base 輪帶
- 使用哪一組掉落表
- 使用哪一組閃電出現權重

### 2. 掉落表

掉落表可以理解為：

- 每次消除後，新符號補進來時的機率表

目前有 3 組：

- A
- B
- C

不同掉落表會影響：

- 後續補進來的符號內容
- 盤面延續能力
- 閃電出現權重對應

### 3. Cascade 流程

只要這一段有得分，就會持續 cascade。

每次 cascade 的順序可理解成：

1. 計算這段盤面得分
2. 標記哪些位置要消除
3. 一般命中格補新符號
4. 金框命中格轉成 Wild
5. 抽這段有幾個閃電
6. 推進乘數等級
7. 把這段得分乘上對應乘數
8. 進入下一段 cascade

### 4. 乘數成長方式

每一段 cascade 的乘數成長來源有兩個：

- 閃電推進
- 該段 cascade 結束後的固定成長

所以只要有連消，乘數通常會一路往上走。

### 5. Base Game 結束條件

當某一段 cascade 已經沒有得分時，base game 結束。

### 6. 免費遊戲觸發條件

base game 結束後，會檢查盤面上的免費遊戲符號數量。

當數量達到：

- `3 個以上`

就會觸發 free game。

## 六、Free Game 邏輯

### 1. 初始免費次數

免費遊戲符號數量決定起始把數：

- 3 個: 10 把
- 4 個: 12 把
- 5 個: 14 把
- 6 個: 16 把

公式可理解為：

- 基本 10 把
- 每多 1 個，再加 2 把

### 2. 每把 free spin 的起始乘數狀態

每一把 free spin 都不是從 base game 的最低乘數開始，而是從較高的乘數階段起跑。

重點：

- 乘數累積存在於單把 free spin 的 cascade 內
- 不會從上一把 free spin 繼承到下一把

### 3. Free Game 的 cascade

free spin 的連消流程與 base game 類似：

- 先算分
- 再消除與補牌
- 金框命中後轉 Wild
- 閃電推進乘數
- 直到該把沒有新得分為止

### 4. 強制閃電機制

free game 有一個額外控制：

- 會指定某一把 free spin
- 如果該把第一段原本沒抽到閃電
- 就強制推進一次乘數

這個機制的目的，是避免整場 free game 過於乾。

### 5. Re-trigger

每一把 free spin 結束後，都會重新檢查盤面上的免費遊戲符號數量。

如果再次達到：

- `3 個以上`

就會追加免費次數。

追加規則和初始觸發相同：

- 3 個加 10 把
- 每多 1 個再加 2 把

### 6. Free Game 結束條件

free game 會在以下任一條件成立時結束：

- 已打完目前所有累積的免費次數
- 已達到免費遊戲總上限

## 七、模式差異

### 1. 一般下注模式

流程：

1. 先跑 base game
2. 檢查是否觸發 free game
3. 若有觸發，再跑整場 free game

### 2. 買免費遊戲模式

重點：

- 成本較高
- 直接使用 free game 對應的盤面與邏輯

這個模式不是改掉 free game 規則，而是改變入口與成本基準。

## 八、統計解讀方式

### 1. RTP

RTP 的概念是：

- 總賠付 / 總下注成本

通常會分開看：

- base game 貢獻
- free game 貢獻
- overall 總和

### 2. 免費遊戲觸發機率

概念：

- 觸發 free game 的 round 數 / 總 round 數

### 3. 免費遊戲平均把數

概念：

- 所有 free spin 總把數 / free game 觸發場數

### 4. 免費遊戲平均倍數

這個指的是：

- 一整場 free game feature 的平均獎金倍數

### 5. 每把 free spin 平均倍數

這個指的是：

- 單把 free spin 的平均獎金倍數

兩者差別要分清楚：

- 前者是「一整場 feature」
- 後者是「feature 裡的一把」

## 九、容易誤解的地方

### 1. 特效核心不是額外炸彈，而是金框轉 Wild

這套規則的延續性，主要來自：

- 金框命中後變 Wild
- 閃電推進乘數

### 2. 免費遊戲乘數不跨把保留

每把 free spin 都會重新從固定較高起點開始，不會接續上一把的乘數等級。

### 3. 買免費遊戲不等於免費

雖然流程直接進 free game，但統計時成本分母也同步提高，因此倍數解讀不能和一般下注直接混看。

## 十、快速結論

這個遊戲的核心體驗可以濃縮成：

- 用 ways 計分
- 透過 cascade 連消延續盤面
- 金框命中後轉 Wild，增加後續連消價值
- 閃電把每段 cascade 的乘數往上推
- base game 收到足夠特殊符號後進入 free game
- free game 從較高乘數起跑，並且可 re-trigger

如果只是要調整數學表或理解結果，最重要的是先分清楚三件事：

- 成本分母怎麼算
- 免費遊戲是以「每場」還是「每把」來看
- 金框與閃電分別影響的是盤面延續性還是乘數成長
