# H026_math_data.xlsx 參數使用說明

## 1. Excel 檔用途總覽

`H026_math_data.xlsx` 是 H026 的數學參數主來源，主要控制：

- 符號與 3 / 4 / 5 連賠付
- 固定線
- BG / FG / BF 各 table 的輪帶內容
- 初始停輪權重
- cascade 掉落權重
- 金框倍率池
- 特殊倍率池觸發率
- BG / FG 掉落表切換

工作表清單：

- `pay_table`
- `value`
- `paylines`
- `BG_strip`
- `BG_strip (2)`
- `BG_strip (3)`
- `FG_strip`
- `FG_strip (2)`
- `FG_strip (3)`
- `BF_strip`
- `weight`

## 2. 工作表：`pay_table`

用途：定義符號 ID 與 3 / 4 / 5 連線賠付。

欄位說明：

| 欄位 | 說明 |
|---|---|
| `Symbol` | 符號名稱 |
| `3` | 3 連賠付 |
| `4` | 4 連賠付 |
| `5` | 5 連賠付 |
| `Id` | 符號編號 |

符號類型：

| Id | Symbol | 類型 |
|---:|---|---|
| 0 | `WW` | Wild |
| 1 | `C1` | Scatter |
| 2~6 | `M1~M5` | 主符號 |
| 7~10 | `A,K,Q,J` | 低分符號 |
| 11~19 | `G1~G5,GA,GK,GQ,GJ` | 金框符號 |

參數用法：

- `WW` 可替代一般付費符號。
- `C1` 不參與一般線獎判斷。
- 金框符號會先映射回底符號再計線。
  - 例如 `G1 -> M1`
  - `GA -> A`
- `pay_table` 只提供基礎線獎，實際派彩還會再乘上最終倍率。

## 3. 工作表：`value`

用途：定義倍率池可抽到的實際倍率值。

欄位說明：

| 欄位 | 說明 |
|---|---|
| `Multiplier_Range` | 倍率值清單 |

目前內容固定為：

`[0, 2, 3, 5, 8, 10, 15, 20, 25, 35, 50, 80, 100]`

參數用法：

- `weight` 工作表中所有倍率權重列，都是對應這 13 個倍率值的位置。
- 程式不是把這欄當區間，而是把它當成可抽中的倍率值清單。
- 金框倍率與特殊倍率池最後都會回到這張表取真正的倍率數字。

## 4. 工作表：`paylines`

用途：定義固定線。

欄位說明：

| 欄位 | 說明 |
|---|---|
| `Line` | 5 碼字串，每碼代表該輪軸取哪一列 |

參數用法：

- H026 盤面是 3 列 5 輪。
- 每碼只能是 `0 / 1 / 2`，分別代表上 / 中 / 下列。
- 程式會逐條讀取 `paylines` 進行線獎判斷。

實際影響：

- 直接影響 hit rate、各符號線獎分布與 RTP。

## 5. 工作表群：`BG_strip`、`BG_strip (2)`、`BG_strip (3)`、`FG_strip`、`FG_strip (2)`、`FG_strip (3)`、`BF_strip`

這 7 張表共同負責：

- 初始停輪出圖
- cascade 補牌時的新符號分布

### 5.1 各工作表代表的 table

| 工作表 | table_id | 場景 |
|---|---:|---|
| `BG_strip` | 0 | BG |
| `BG_strip (2)` | 1 | BG |
| `BG_strip (3)` | 2 | BG |
| `FG_strip` | 3 | FG |
| `FG_strip (2)` | 4 | FG |
| `FG_strip (3)` | 5 | FG |
| `BF_strip` | 6 | Feature Buy |

### 5.2 共用欄位結構

| 欄位群 | 說明 |
|---|---|
| `Symbol_ID_R1 ~ Symbol_ID_R5` | 各輪軸的符號序列 |
| `Strip_Weight_R1 ~ Strip_Weight_R5` | 初始停輪位置權重 |
| `Drop_Weight_A_R1 ~ Drop_Weight_A_R5` | 掉落表 A 的補牌權重 |
| `Drop_Weight_B_R1 ~ Drop_Weight_B_R5` | 掉落表 B 的補牌權重 |

### 5.3 `Symbol_ID_R1 ~ Symbol_ID_R5`

用途：定義每輪 strip 的符號內容。

參數用法：

- 抽中停輪位置後，程式從該位置往下取 3 格形成盤面。
- 金框符號在生成盤面時會被標記為金框，但計線時仍會回到其底符號。

實際影響：

- 控制 wild、scatter、金框與一般符號的自然出現率。

### 5.4 `Strip_Weight_R1 ~ Strip_Weight_R5`

用途：控制初始停輪位置被選中的機率。

實際影響：

- 改變相同 strip 中不同區段的出現頻率。
- 直接影響初始盤面型態。

### 5.5 `Drop_Weight_A_R1 ~ Drop_Weight_A_R5`

用途：當本局選到掉落表 A 時，控制 cascade 補進來的符號分布。

參數用法：

- 補牌不是沿原 strip 下落。
- 而是直接依 `Drop_Weight_A` 對符號 ID 做權重抽選。
- 每輪用各自的 `R1~R5` 權重。

### 5.6 `Drop_Weight_B_R1 ~ Drop_Weight_B_R5`

用途：和 `Drop_Weight_A` 相同，但對應掉落表 B。

### 5.7 strip 表的流程影響

- 初始盤面來源：`Symbol_ID_*` + `Strip_Weight_*`
- 消除後補牌來源：`Drop_Weight_A_*` 或 `Drop_Weight_B_*`
- BG / FG / BF 的 table 切換，會改變使用哪一張 strip 表

## 6. 工作表：`weight`

用途：集中管理 table 切換、倍率池與掉落表切換。

### 6.1 主要欄位群總覽

| 欄位群 | 用途 |
|---|---|
| `Table_BG` | BG 三張 table 的選擇權重 |
| `Table_FG` | FG table 權重，目前主流程未直接使用 |
| `Table_BF` | BF table 權重 |
| `Eliminate_Table_Weight_BG` | BG 掉落表 A / B 切換權重 |
| `Eliminate_Table_Weight_FG` | FG 掉落表 A / B 切換權重 |
| `Multiple_Selection_Weight_Special_*` | 特殊倍率池權重 |
| `Multiple_Selection_Weight_R3_Before_*` | 初始盤第 3 輪金框倍率權重 |
| `Multiple_Selection_Weight_Before_*` | 初始盤其他輪金框倍率權重 |
| `Multiple_Selection_Weight_R3_After_*` | cascade 後第 3 輪新金框倍率權重 |
| `Multiple_Selection_Weight_After_*` | cascade 後其他輪新金框倍率權重 |
| `Used_Special_Pool_Weight_*` | BG 特殊倍率池觸發率 |

### 6.2 `Table_BG`

用途：決定 BG 使用 `BG_strip / BG_strip (2) / BG_strip (3)` 的機率。

參數用法：

- 只使用前 3 列。
- 3 列分別對應 `table_id 0 / 1 / 2`。

### 6.3 `Table_FG`

用途：名義上是 FG table 權重。

目前主流程現況：

- 這組欄位有載入。
- 但實際 FG 不會直接抽 `Table_FG`。
- FG 會依累積倍率和強制切表：
  - `< 10` 用 `FG_strip`
  - `< 20` 用 `FG_strip (2)`
  - `>= 20` 用 `FG_strip (3)`

因此：

- `Table_FG` 目前屬於「有載入，但未直接參與主流程」的欄位。

### 6.4 `Table_BF`

用途：決定 Feature Buy 使用哪張 BF table。

參數用法：

- 程式寫法保留了 3 個 BF table 的空間。
- 但目前 Excel 只有 `BF_strip` 1 張實體表。
- 目前第 1 列有值、後兩列為 0，所以實際只會落到 `BF_strip`。

### 6.5 `Eliminate_Table_Weight_BG`

用途：決定 BG 每局使用掉落表 A 或 B。

參數用法：

- 只使用前 2 列。
- 第 1 列對應 A
- 第 2 列對應 B

### 6.6 `Eliminate_Table_Weight_FG`

用途：決定 FG 每局使用掉落表 A 或 B。

用法與 `Eliminate_Table_Weight_BG` 相同，但作用在 FG。

### 6.7 `Multiple_Selection_Weight_Special_*`

用途：特殊倍率池命中時，決定該金框抽到哪個倍率。

參數用法：

- 每列對應 `value.Multiplier_Range` 的同索引倍率值。
- 這組只在 BG 特殊池命中時使用。

### 6.8 `Multiple_Selection_Weight_R3_Before_*`

用途：初始盤面中，第 3 輪金框的倍率權重。

### 6.9 `Multiple_Selection_Weight_Before_*`

用途：初始盤面中，非第 3 輪金框的倍率權重。

### 6.10 `Multiple_Selection_Weight_R3_After_*`

用途：cascade 補進來的新金框，若位於第 3 輪時的倍率權重。

### 6.11 `Multiple_Selection_Weight_After_*`

用途：cascade 補進來的新金框，若不在第 3 輪時的倍率權重。

### 6.12 `Used_Special_Pool_Weight_*`

用途：決定 BG 是否觸發特殊倍率池。

參數用法：

- 這組欄位不是倍率權重，而是特殊池觸發率。
- 列索引不是倍率，而是當局金框數量。
- 程式使用 `min(gold_count, 9)` 取列：
  - 第 0 列對應金框數 0
  - 第 1 列對應金框數 1
  - ...
  - 第 9 列對應金框數 9 以上

實際影響：

- 只有 BG 會使用特殊倍率池判斷。
- FG 與 BF 不走這套特殊池邏輯。

## 7. BG 流程對應

### 7.1 BG 開局流程

BG 每局的主要參數流程如下：

1. 先用 `weight.Table_BG` 選出 BG table。
2. 再用該 strip 的 `Strip_Weight_R1~R5` 抽停輪位置。
3. 依 `Symbol_ID_R1~R5` 生成初始盤面。
4. 盤面上的金框會先判定是否觸發 `Used_Special_Pool_Weight_*`。
5. 若命中特殊池，該金框用 `Multiple_Selection_Weight_Special_*` 抽倍率。
6. 若沒命中特殊池：
   - 第 3 輪金框用 `Multiple_Selection_Weight_R3_Before_*`
   - 其他輪金框用 `Multiple_Selection_Weight_Before_*`
7. 再用 `Eliminate_Table_Weight_BG` 決定本局是掉落表 A 或 B。

### 7.2 BG cascade 流程

BG 發生消除後：

- 一般命中符號會移除。
- 命中的金框會把自身倍率加進最終倍率和，並轉成 `WW` 留在盤面。
- 新補進來的符號由 `Drop_Weight_A_*` 或 `Drop_Weight_B_*` 決定。
- 若新補進來的是金框：
  - 第 3 輪用 `Multiple_Selection_Weight_R3_After_*`
  - 其他輪用 `Multiple_Selection_Weight_After_*`

### 7.3 BG 與 FG 觸發關係

- BG 盤面上的 `C1` 數量若達 3 個以上，就觸發 FG。
- FG 次數 = `15 + (scatter_count - 3) * 2`

## 8. FG 流程對應

### 8.1 FG table 切換規則

FG 的 table 不是靠 `Table_FG` 權重抽，而是依 FG 累積倍率和切換：

- 累積倍率和 `< 10`：用 `FG_strip`
- 累積倍率和 `< 20`：用 `FG_strip (2)`
- 累積倍率和 `>= 20`：用 `FG_strip (3)`

### 8.2 FG 開局流程

FG 每局的主要參數流程如下：

1. 依當前 `fg_multiplier_sum` 選擇 FG table。
2. 用該 table 的 `Strip_Weight_R1~R5` 與 `Symbol_ID_R1~R5` 生成盤面。
3. 初始金框倍率不走 BG 特殊池邏輯。
4. 初始金框倍率分配：
   - 第 3 輪用 `Multiple_Selection_Weight_R3_Before_*`
   - 其他輪用 `Multiple_Selection_Weight_Before_*`
5. 用 `Eliminate_Table_Weight_FG` 決定本局掉落表 A 或 B。

### 8.3 FG cascade 與倍率累加

FG 消除後：

- 命中的金框一樣會把倍率加到 `fg_multiplier_sum`。
- 該累積值會帶到下一個 FG spin，作為下一局 table 切換條件。
- 新補進來金框的倍率來源：
  - 第 3 輪：`Multiple_Selection_Weight_R3_After_*`
  - 其他輪：`Multiple_Selection_Weight_After_*`

### 8.4 FG 再觸發

- FG 內若再次出現 `C1 >= 3`，會再加 `15 + (scatter_count - 3) * 2`
- 但總剩餘 FG 次數會被 `fg_spin_cap = 50` 封頂
