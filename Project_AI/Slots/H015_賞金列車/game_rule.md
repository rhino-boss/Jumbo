## §2. 盤面結構

```text
         R1   R2   R3   R4   R5   R6
         ↓    ↓    ↓    ↓    ↓    ↓
      ┌────┬────┬────┬────┬────┬────┐
      │    │    │    │    │    │    │
      ├────┼────┼────┼────┼────┼────┤
      │    │    │    │    │    │    │
      ├────┼────┼────┼────┼────┼────┤
      │    │    │    │    │    │    │
           ├────┼────┼────┼────┤
           │    │    │    │    │
                ├────┼────┤
                │    │    │
```

文字說明：

* 主盤面共有 6 輪顯示欄位，實際有效格為 24 格。
* 各輪有效高度固定為 `3 / 4 / 5 / 5 / 4 / 3`，所以總 Ways 固定為 `3 × 4 × 5 × 5 × 4 × 3 = 3600`。
* 中獎採由左至右連續計算；當某一輪沒有該符號或 Wild 時，該符號的 Ways 中止。
* 本作沒有獨立 Extra Reel。

## §3. 符號類別

### 3.1 一般符號

| 類別 | 符號 |
| --- | --- |
| 高分符號（M） | `M1`、`M2`、`M3`、`M4` |
| 低分符號（L） | `A`、`K`、`Q`、`J` |

寫明：

* 目前主來源資料未配置 Block / Fat Symbol；所有符號皆以 1×1 計算。
* `G1 ~ G4`、`GA ~ GJ` 為上述一般符號的金框版本，判獎時先還原成對應的一般符號。
* 金框符號只出現在 `R3`、`R4`。

### 3.2 特殊符號

| 符號 | 出現位置 | 行為 |
| --- | --- | --- |
| **WW / Wild** | 不自然出現在初始輪帶；僅由中獎金框轉化而成 | 替代一般符號，參與 Ways 計算 |
| **C1 / Scatter** | 6 輪皆可出現 | 不參與 Ways 中獎；於整把連消全部結束後統計總數，用來觸發或再觸發 Free Game |
| **C2** | 目前主來源未配置於可見輪帶 | 保留特殊符號代號，現行數學模型未作為實際盤面玩法使用 |

## §4. 賠率表（Pay Table）

**單位：倍 / Bet × 該符號的 Ways 數量**

| 符號 | 6 連 | 5 連 | 4 連 | 3 連 |
| --- | --- | --- | --- | --- |
| `M1` / `G1` | 2.5 | 1.5 | 1.0 | 0.5 |
| `M2` / `G2` | 1.5 | 1.0 | 0.75 | 0.4 |
| `M3` / `G3` | 1.0 | 0.75 | 0.5 | 0.25 |
| `M4` / `G4` | 1.0 | 0.75 | 0.5 | 0.25 |
| `A` / `GA` | 0.5 | 0.3 | 0.2 | 0.1 |
| `K` / `GK` | 0.5 | 0.3 | 0.2 | 0.1 |
| `Q` / `GQ` | 0.25 | 0.15 | 0.1 | 0.05 |
| `J` / `GJ` | 0.25 | 0.15 | 0.1 | 0.05 |

補充說明：

* `WW`、`C1`、`C2` 不提供一般 Ways 派彩。
* 實際派彩公式為：`符號賠率 × Ways 數量 × Bet`。
* 同一輪若有多種符號同時成立 Ways，會各自獨立計算後相加。

## §5. 核心遊戲流程（Base Game）

1. 玩家選擇押注並開始 Spin。
2. 系統依 BG table 權重抽出當局使用的初始輪帶表。
3. 盤面生成後，先把非有效記分區的位置排除，只保留固定 24 格的有效區。
4. 若盤面中有金框符號，先將其還原成一般符號參與 Ways 計算，但保留金框位置標記。
5. 系統依當局設定抽出本次要採用的掉落表類型：`A`、`B` 或 `C`。
6. 每次 Cascade 開始前，先抽出本段要出現幾個「炸彈階數」。
7. 盤面由左至右進行 Ways 判獎；Wild 可替代一般符號。
8. 若有一般符號中獎，先計算該段原始贏分。
9. 參與中獎的普通符號會在掉落時補入新符號；參與中獎的金框符號不直接消失，而是轉成 Wild 留在盤面上。
10. 該段贏分會套用目前乘數階級對應的倍數值後，再累加到本局總得分。
11. 掉落補牌後若再形成新的 Ways，中獎、轉 Wild、掉落、乘倍流程會持續重複。
12. 直到盤面不再形成新中獎後，才統計最終盤面上的 `C1 / Scatter` 數量。
13. 若最終盤面 `C1 >= 3`，則觸發 Free Game。

補充說明：

* 本作的乘數不是直接來自盤面上的可見特殊符號，而是由每段 Cascade 抽出的「炸彈階數」推進。
* 每次 Cascade 結束後，乘數階級除了受炸彈影響外，還會再因該次消除額外向上推進 1 階。

## §6. 核心特色

### 6.1 金框 → Wild

* 金框符號只出現在 `R3`、`R4`。
* 金框符號在判獎時先視為其對應的一般符號。
* 若該顆金框實際參與本次中獎，則該位置在結算後不會補新符號，而是直接轉成 `WW / Wild`。
* 該 Wild 會留在盤面上，參與後續 Cascade 的 Ways 計算。
* Wild 不自然掉落，只能由金框轉化而來。

### 6.2 炸彈乘數階梯

* 每段 Cascade 都會先抽出本段的炸彈數量，用來推進乘數階級。
* 乘數階梯目前為：`x1`、`x2`、`x4`、`x8`、`x16`、`x32`、`x64`、`x128`、`x256`、`x512`、`x1024`。
* Base Game 的乘數階級從 `x1` 起算。
* Free Game 的乘數階級從較高階開始，現行程式邏輯為從 `x8` 對應的階級起算。
* 每段實際贏分使用的是「炸彈推進後」當下對應的乘數值。

範例：

* 若 Base Game 第一次消除前抽到 `2` 個炸彈，且該次有得分，則第一次得分使用 `x4`。
* 第一次消除結束後，乘數階級會再因為完成一次消除而額外往上推進 1 階。
* 若第二次消除前再抽到 `1` 個炸彈，且第二次也有得分，則第二次得分使用 `x16`。
* 若第二次消除後沒有再形成新中獎，則流程在該次結算後結束，不會再有下一段倍數結算。

### 6.3 掉落表 A / B / C

* 每把 Spin 開始時，系統會先決定當把採用 `A`、`B` 或 `C` 哪一套掉落表。
* 之後整把 Spin 的補牌都沿用該套掉落類型。
* 連消前幾段與後續 4 段以上的補牌權重分開配置，因此不同 Cascade 深度的掉落內容可能不同。

## §7. 免費遊戲（Free Spins / Free Game, FG）

### 7.1 觸發

* Base Game 所有 Cascade 結束後，若最終盤面 `C1 >= 3`，即可觸發 Free Game。
* 初始場次公式為：`10 + (Scatter 數量 - 3) × 2`。
* 目前主來源邏輯是依最終盤面 `C1` 總數直接帶入公式，未另外限制只計某幾顆或某幾輪。

### 7.2 FG 玩法

* FG 使用專用 table 權重與專用掉落權重。
* FG 的乘數階級起點高於 BG，現行邏輯為從 `x8` 階級開始。
* 每段 Cascade 同樣會先抽炸彈，再以當下乘數值放大該段贏分。
* 目前程式包含一個 FG 保底機制：若指定的 FG 場次在首段 Cascade 未抽到炸彈，會強制補 1 階炸彈。
* 單次進入 FG 後，包含 retrigger 在內，總場數上限為 `50`。

### 7.3 Retrigger（加局）

* FG 單局結束後，若最終盤面 `C1 >= 3`，可再次加局。
* 加局公式與初始觸發相同：`10 + (Scatter 數量 - 3) × 2`。
* FG 最後一場若再次達成觸發條件，仍可繼續加局，但總場數不超過 50。

---

## §8. Buy Feature（購買特色）

* `Feature Buy` 價格為 `75x Bet`。
* Buy 模式會改用 BF 進場輪帶，目標是直接進入一輪可觸發 FG 的進場局面。
* 進入 FG 後，仍沿用本作現行 FG 核心規則。
* 目前主來源程式中，Buy 進入後的 FG table 會依 BF 專用權重決定使用哪組 FG 配置。

## §9. 邊界 / 例外情境（Edge Cases）

### 9.1 金框 / Wild 相關

* **9.1.1 金框只出現在 R3、R4**：R1、R2、R5、R6 不會自然生成金框。
* **9.1.2 金框未中獎不轉換**：只有實際參與本段中獎的金框才會轉成 Wild。
* **9.1.3 Wild 不自然出現**：Wild 不存在於初始輪帶與掉落表中，只能由中獎金框轉換。
* **9.1.4 Wild 不保留金框屬性**：金框一旦轉成 Wild，後續不再視為金框符號。

### 9.2 倍數機制相關

* **9.2.1 倍數結算時機**：每段 Cascade 的原始贏分會以該段當下的乘數值放大後再入帳。
* **9.2.2 倍數來源不是盤面符號**：現行 H015 並沒有獨立的可見炸彈符號；乘數推進來自每段額外抽選的炸彈階數。
* **9.2.3 Cascade 會再推進乘數階級**：即使該段沒有抽到額外炸彈，完成該次消除後，後續階級仍會再向上推進 1 階。
* **9.2.4 FG 高起始階級**：FG 不是從 `x1` 開始，而是從較高乘數階級起步。

### 9.3 Scatter 相關

* **9.3.1 Scatter 可出現在 6 輪**：主來源輪帶顯示 6 輪皆可出現 `C1`。
* **9.3.2 Scatter 不參與 Ways**：`C1` 不提供一般派彩。
* **9.3.3 Scatter 不參與消除**：一般符號連消過程中，Scatter 會留到整把結束後再做 FG 判定。
* **9.3.4 觸發判定點在整把結束後**：不是看到 `C1` 就立即進 FG，而是要等所有 Cascade 完成。

### 9.4 Free Game 相關

* **9.4.1 FG 可 retrigger**：觸發條件與計算公式與 BG 結束後的 FG 觸發相同。
* **9.4.2 FG 總場數上限 50**：包含 retrigger 在內一律受上限限制。
* **9.4.3 FG 存在保底炸彈機制**：現行主來源邏輯中，指定 FG 場次若首消未出炸彈，會補 1 階炸彈。

### 9.5 Buy Feature 相關

* **9.5.1 Buy 價格固定**：`75x Bet`。
* **9.5.2 Buy 進入後仍依本作 FG 規則處理**：進場模式不同，但後續核心玩法仍是同一套 FG 邏輯。

### 9.6 Way Game 計算相關

* **9.6.1 Ways 由左至右連續計算**：某輪若沒有該符號或 Wild，該符號 Ways 結束。
* **9.6.2 Wild 替代不改變 Ways 算法**：Wild 視為 1 個對應符號參與該輪數量計算。
* **9.6.3 多種符號可同時中獎**：各符號獨立算 Ways 與派彩後相加，再套用當段乘數值。

### 9.7 連消（Cascade）相關

* **9.7.1 Cascade 終止條件**：直到該把 Spin 不再形成新中獎為止。
* **9.7.2 補位來源為掉落表**：一般中獎格不沿用原輪帶往下補，而是直接依當把選定的掉落表補入新符號。
* **9.7.3 掉落表按連消深度切換**：前 3 次連消與第 4 次之後使用的掉落權重段位不同。
