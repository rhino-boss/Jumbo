# H026_彩罐熱舞 製作流程

## 目的

這份文件用來整理：如果要開一個新的 slot 專案，參考 `Project_AI` 底下既有 `.md` 與 `H026_彩罐熱舞` 範例時，應該先看什麼、再產出什麼，並說明 `H026_彩罐熱舞` 在排除 `Source` 後各檔案的角色。

---

## 一、先看哪些參考文件

### 1. 共用規範

1. `Project_AI/slot_demogame_standards.md`
   - 用途：定義 `Demogame` 應有哪些 UI 區塊、互動、資料綁定、RNG 顯示、最低驗收標準。
   - 影響輸出：`index.html`、部分 `config.js` 欄位需求。

2. `Project_AI/slot_simulator_standards.md`
   - 用途：定義 `Simulator.py` 的程式結構、前段設定區、Numba 核心、console 顯示、xlsx 報表輸出格式。
   - 影響輸出：`Simulator.py`、`Record/*.xlsx`。

### 2. 專案規格文件

3. `Project_AI/H026_彩罐熱舞/game_rule_pinata bonanza.md`
   - 用途：定義玩法規則。
   - 內容包含：盤面、symbol、paytable、cascade、gold -> wild、multiplier、FG、Buy Feature、edge cases。
   - 影響輸出：`config.js` 欄位設計、`Simulator.py` 邏輯、`index.html` 顯示內容。

4. `Project_AI/H026_彩罐熱舞/Tool/xlsx_config_usage_mapping.md`
   - 用途：說明 `xlsx -> config.js -> simulator/index.html` 的資料流。
   - 影響輸出：`config.js` 生成器、`Simulator.py` 與 `index.html` 的欄位對接方式。

## 二、H026 排除 Source 後，各檔案是什麼

### 1. 規格 / 說明類

1. `game_rule_pinata bonanza.md`
   - 手工整理的規則文件。

2. `Tool/xlsx_config_usage_mapping.md`
   - 手工整理的欄位對照文件。

### 2. 生成 / 轉換工具類

3. `Tool/xlsx_to_config.py`
   - 讀取 `Source` 內的數學 `xlsx`，輸出 `config.js`。
   - 屬於 generator，不是 runtime 主程式。

4. `update_config.bat`
   - 包裝 `xlsx_to_config.py` 的更新入口。
   - 用來把最新 `xlsx` 重新轉成 `config.js`。

### 3. Runtime / 成品類

5. `config.js`
   - 中央資料檔。
   - `Simulator.py` 與 `index.html` 都讀它。
   - 原則上應由 `xlsx_to_config.py` 生成，不建議直接手改。

6. `Simulator.py`
   - 模擬器主程式。
   - 讀 `config.js`，執行單局 debug 或大量模擬，並輸出報表。

7. `index.html`
   - 離線 demogame。
   - 讀 `config.js`，用同一套資料結構展示 UI、盤面、流程與狀態。

### 4. 輸出結果類

8. `Record/*.xlsx`
   - `Simulator.py` 執行後產出的報表。

### 5. 可忽略類

9. `__pycache__`
   - Python 執行快取，不屬於專案交付物。

## 三、新專案建議製作順序

### Step 1. 先整理玩法規則文件

先產出一份新的 `game_rule_xxx.md`。

參考來源：

1. 遊戲提案文件
2. Help 截圖 / 競品說明
3. 若資料衝突，先決定主來源

這一步要定清楚：

1. 盤面規格
2. symbol 定義
3. paytable
4. line / ways / cluster 判定方式
5. base game 流程
6. feature / multiplier / wild / scatter 規則
7. free game / respin / buy feature
8. edge cases

輸出：

1. `game_rule_xxx.md`

### Step 2. 確認數學資料 xlsx 結構

先確認數學 `xlsx` 是否能支撐 runtime 需要的欄位。

參考來源：

1. `Tool/xlsx_config_usage_mapping.md`
2. `config.js` 現有欄位
3. `slot_demogame_standards.md`
4. `slot_simulator_standards.md`

這一步要確認：

1. `overview`
2. `pay_table`
3. `paylines`
4. `value`
5. `weight`
6. 各場景 strip sheet
7. 是否有足夠資料支援 feature 規則

輸出：

1. 新專案的數學 `xlsx`
2. 如有需要，補一份新的 `xlsx_config_usage_mapping.md`

### Step 3. 寫 xlsx -> config 的轉換器

先做 generator，再做 runtime。

參考來源：

1. `Tool/xlsx_to_config.py`
2. `Tool/xlsx_config_usage_mapping.md`

這一步要做：

1. 定義哪些欄位直接來自 `xlsx`
2. 定義哪些欄位是衍生欄位
3. 定義哪些欄位是靜態常數或 generator 補值
4. 保留 template merge 機制，避免每次重生都覆蓋靜態設定

輸出：

1. `Tool/xlsx_to_config.py`
2. `update_config.bat`
3. `config.js`

### Step 4. 先生成 config.js

H026 的實際流程已經固定為：

1. 提供 `xlsx`
2. 用 `update_config.bat`
3. `update_config.bat` 呼叫 `Tool/xlsx_to_config.py`
4. 生成或更新 `config.js`

這一步完成後，`config.js` 應成為 simulator 與 demogame 的共同資料來源。

輸出：

1. `config.js`

### Step 5. 開發 Simulator.py

參考來源：

1. `slot_simulator_standards.md`
2. `game_rule_xxx.md`
3. `config.js`

這一步要做：

1. 前段設定區
2. config 載入
3. 盤面生成
4. 評獎
5. feature 流程
6. cascade / FG / buy feature
7. console 固定結果區
8. console by-game 區
9. xlsx 報表輸出

輸出：

1. `Simulator.py`
2. `Record/*.xlsx`

### Step 6. 開發 index.html

參考來源：

1. `slot_demogame_standards.md`
2. `game_rule_xxx.md`
3. `config.js`

這一步要做：

1. Header / Hero
2. Feature 狀態列
3. Reel Board
4. Message Bar
5. Buy / Extra Bet 區
6. Player Info
7. 主控制列
8. Session 統計
9. RNG 區
10. Line Wins
11. Spin Result

重點：

1. `index.html` 和 `Simulator.py` 要共用同一份 `config.js`
2. UI 顯示規則不能脫離 `game_rule_xxx.md`
3. table / reel / multiplier 顯示要與 config 欄位一致

輸出：

1. `index.html`

### Step 7. 驗證資料流一致

參考來源：

1. `Tool/xlsx_config_usage_mapping.md`
2. `config.js`
3. `Simulator.py`
4. `index.html`

要確認：

1. `xlsx` 的主要欄位都有正確進 `config.js`
2. `Simulator.py` 與 `index.html` 讀的是同一份欄位定義
3. 規則文件、數學資料、runtime 行為三者一致

輸出：

1. 更新版 `xlsx_config_usage_mapping.md`
2. 修正後的 `config.js` / `Simulator.py` / `index.html`

## 四、最推薦的實作依賴順序

不要直接先寫 `Simulator.py` 或 `index.html`，建議依下面順序：

1. `game_rule_xxx.md`
2. 數學 `xlsx`
3. `Tool/xlsx_to_config.py`
4. `config.js`
5. `Simulator.py`
6. `index.html`
7. `Tool/xlsx_config_usage_mapping.md`
8. `Record/*.xlsx` 驗證輸出

原因：

1. `game_rule` 決定玩法邏輯
2. `xlsx` 決定數學輸入
3. `config.js` 是 simulator 與 demogame 的共同契約
4. `Simulator.py` 與 `index.html` 都應建立在共同契約之上

## 六、最短版本流程

如果只看最短執行順序，就是：

1. 先寫規則文件
2. 準備數學 `xlsx`
3. 寫 `xlsx_to_config.py`
4. 用 `update_config.bat` 生成 `config.js`
5. 讓 `Simulator.py` 與 `index.html` 共讀 `config.js`
6. 跑 simulator 輸出 `Record/*.xlsx`
7. 回頭補齊欄位對照文件與驗證
